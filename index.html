Methodology

Why DBSCAN fits transport usage data

College transport data often contains dense student pickup zones, lightly used routes, and unusual trips caused by events, exams, or temporary schedule changes. DBSCAN is useful because it can discover dense groups without forcing every data point into a cluster.

01

Collect

Gather route, stop, boarding time, student count, and distance information.

02

Prepare

Clean missing values, normalize numeric fields, and encode route categories.

03

Cluster

Apply DBSCAN and K-Means to compare density-driven and centroid-driven grouping.

04

Interpret

Identify crowded stops, underused paths, unusual usage, and scheduling opportunities.

DBSCAN versus K-Means

Factor DBSCAN K-Means

Project conclusion

DBSCAN is better for discovering natural transport demand groups and identifying unusual student travel records. K-Means is useful as a comparison model when the project needs simple, fixed segments for reporting.

  • Use DBSCAN for anomaly-aware route planning.
  • Use K-Means for quick cluster summaries.
  • Compare both models using silhouette score and domain interpretation.
